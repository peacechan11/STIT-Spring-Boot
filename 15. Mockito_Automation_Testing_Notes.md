# Mockito in Automation Testing

## Mockito ဘာလို့ လိုအပ်သလဲ?

Automation Testing ကောင်းတစ်ခု ဖြစ်ဖို့ အောက်ပါ အချက်များကို ဖြည့်ဆည်းနိုင်ရပါမယ်။

- **Fast** – Test များကို လျင်မြန်စွာ Run နိုင်ရမည်
- **Reliable** – အမြဲတမ်း တူညီသော Result များ ရရှိရမည်
- **Independent** – External Systems များအပေါ် မမှီခိုသင့်ပါ

ထို့ကြောင့် Test များကို Run လုပ်ရာတွင် Database, API, Network စသည့် External Dependencies များအပေါ် မူတည်နေခြင်းကို လျှော့ချရန် လိုအပ်ပါသည်။

## Mockito အသုံးပြုခြင်း၏ အကျိုးကျေးဇူးများ

Mockito ကို အသုံးပြုခြင်းဖြင့် Real Objects များအစား Fake Objects (Mock Objects) များကို ဖန်တီးပြီး Test ပြုလုပ်နိုင်ပါသည်။

ဥပမာ -

- **Database Down?** → No Problem
- **Internet Down?** → No Problem
- **API Not Ready?** → No Problem

External Services များ အလုပ်မလုပ်သေးသော်လည်း Application Logic ကို သီးခြားစမ်းသပ်နိုင်ပါသည်။

## Mockito ဆိုတာ ဘာလဲ?

**Mockito** သည် Java Ecosystem တွင် Mock Objects များ ဖန်တီးရန် အကျော်ကြားဆုံး Testing Framework များထဲမှ တစ်ခု ဖြစ်ပါသည်။

Mockito ကို အသုံးပြုခြင်းဖြင့် Unit Tests များကို ပိုမို မြန်ဆန်၊ တိကျပြီး ယုံကြည်စိတ်ချရစွာ ရေးသားနိုင်ပါသည်။

## အသုံးများသော Annotations များ

### @Mock

Fake Object (Mock Object) တစ်ခုကို ဖန်တီးပေးပါသည်။

```java
@Mock
private UserRepository userRepository;
```

### @InjectMocks

Mock Objects များကို Test လုပ်မည့် Class အတွင်းသို့ Automatic Inject လုပ်ပေးပါသည်။

```java
@InjectMocks
private UserService userService;
```

## Summary

Mockito သည် Automation Testing နှင့် Unit Testing များတွင် External Dependencies များကို ဖယ်ရှားပြီး Test များကို

- ပိုမို မြန်ဆန်စေခြင်း
- ပိုမို ယုံကြည်စိတ်ချရစေခြင်း
- Environment များအပေါ် မမှီခိုစေခြင်း

တို့အတွက် အရေးကြီးသော Framework တစ်ခု ဖြစ်ပါသည်။
