# 💉 Dependency Injection (DI) in Spring Boot

---

## 📌 What is Dependency Injection?

Dependency Injection (DI) ဆိုတာ  
👉 Class တစ်ခုအတွက်လိုအပ်တဲ့ dependency (object) ကို  
👉 ကိုယ်တိုင် `new` နဲ့ create မလုပ်ဘဲ  
👉 Spring IoC Container က create လုပ်ပြီး inject လုပ်ပေးတာဖြစ်ပါတယ်။

---

## 🧠 Simple Idea

❌ Without DI

```java
OrderService orderService = new OrderService();
```

👉 Tight coupling ❗

---

✅ With DI

```java
@Autowired
private OrderService orderService;
```

👉 Spring inject လုပ်ပေးတယ် ✔️

---

# 🧩 Sample Code

## 1. Interface

```java
public interface PaymentService {

}
```

---

## 2. Implementation

```java
@Service
public class KBZPayService implements PaymentService {

    public void pay() {

    }

```

# 💡 Types of Dependency Injection

---

## 1. Constructor Injection ⭐ (Best Practice)

```java
@Service
public class OrderService {

    private final PaymentService paymentService;

    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }
}
```

✔️ Recommended  
✔️ Immutable  
✔️ Easy testing  

---

## 2. Field Injection

```java
@Service
public class OrderService {

    @Autowired
    private PaymentService paymentService;
}
```

⚠️ Not recommended  
- Hard to test  
- Reflection-based  

---

## 3. Setter Injection

```java
@Service
public class OrderService {

    private PaymentService paymentService;

    @Autowired
    public void setPaymentService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }
}
```

✔️ Optional dependency  
✔️ Can change later  

---

## 🔄 Example Usage

```java
@Service
public class OrderService {

    private final PaymentService paymentService;

    public OrderService(PaymentService paymentService) {
        this.paymentService = paymentService;
    }

    public void placeOrder() {
        paymentService.pay();
    }
}
```

---

## 🚀 Benefits of DI

- Loose coupling  
- Easy testing (mocking)  
- Flexible implementation  
- Clean architecture  

---

## ❌ Tight Coupling vs ✅ Loose Coupling

### Tight Coupling

```java
private PaymentService paymentService = new KBZPayService();
```

---

### Loose Coupling

```java
private final PaymentService paymentService;

public OrderService(PaymentService paymentService) {
    this.paymentService = paymentService;
}
```
