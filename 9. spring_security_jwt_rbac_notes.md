# Spring Boot Security & Authentication Notes

## 1. Spring Security ဆိုတာဘာလဲ?

Spring Security ဆိုတာ **Spring Boot application ကို လုံခြုံရေးကာကွယ်ပေးတဲ့ framework** ပါ။

အလွယ်ဆုံးပြောရရင်:

```text
Spring Boot App = အိမ်
Spring Security = တံခါး + လော့ခ် + security guard
```

Spring Security က ဒီလိုအရာတွေကို handle လုပ်ပေးတယ်။

- Login စစ်တာ
- User ဘယ်သူလဲ verify လုပ်တာ
- ဘယ် user က ဘာလုပ်ခွင့်ရှိလဲ စစ်တာ
- Password ကို hash လုပ်တာ
- API တွေကို unauthorized access ကနေကာကွယ်တာ
- Role/Permission အလိုက် access control လုပ်တာ

---

## 2. Authentication ဆိုတာဘာလဲ?

**Authentication** ဆိုတာ user က တကယ်ဘယ်သူလဲဆိုတာ စစ်တာပါ။

ဥပမာ:

```text
email = user@gmail.com
password = 123456
```

System က username/password မှန်လား စစ်တယ်။

```text
မှန် → Login success
မမှန် → Login failed
```

အလွယ်မှတ်ရန်:

```text
Authentication = ဘယ်သူလဲ စစ်တာ
```

---

## 3. Authorization ဆိုတာဘာလဲ?

**Authorization** ဆိုတာ login ဝင်ပြီးတဲ့ user က ဘာလုပ်ခွင့်ရှိလဲ စစ်တာပါ။

ဥပမာ:

```text
ADMIN → user delete လုပ်ခွင့်ရှိ
USER  → profile ကြည့်ခွင့်ပဲရှိ
```

အလွယ်မှတ်ရန်:

```text
Authorization = ဘာလုပ်ခွင့်ရှိလဲ စစ်တာ
```

---

## 4. Authentication vs Authorization

| Concept | Meaning | Example |
|---|---|---|
| Authentication | User ဘယ်သူလဲ စစ်တာ | Login username/password စစ်တာ |
| Authorization | User ဘာလုပ်ခွင့်ရှိလဲ စစ်တာ | ADMIN ပဲ delete လုပ်ခွင့်ရှိ |

အလွယ်မှတ်ရန်:

```text
Authentication = Login စစ်တာ
Authorization = Permission စစ်တာ
```

---

## 5. Spring Security Request Flow

Spring Security က request တိုင်းကို controller မရောက်ခင် စစ်ပေးတယ်။

```text
Client Request
      ↓
Spring Security Filter Chain
      ↓
Authentication စစ်
      ↓
Authorization စစ်
      ↓
Controller / API
```

ဥပမာ:

```text
User → GET /api/profile
        ↓
Spring Security က token ရှိလားစစ်
        ↓
Token မှန်လားစစ်
        ↓
Role မှန်လားစစ်
        ↓
OK ဆို Controller ကိုပို့
```

## 6. Important Spring Security Components

### 6.1 SecurityFilterChain

`SecurityFilterChain` က Spring Security ရဲ့ main security configuration ပါ။

ဘယ် API က public ဖြစ်မလဲ၊ ဘယ် API က login လိုမလဲ ဆိုတာ configure လုပ်တယ်။

```java
@Bean
public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
    return http
        .csrf(csrf -> csrf.disable())
        .authorizeHttpRequests(auth -> auth
            .requestMatchers("/api/auth/**").permitAll()
            .anyRequest().authenticated()
        )
        .build();
}
```

Meaning:

```text
/api/auth/** → public
အခြား API အကုန် → login လုပ်မှရ
```

---

### 6.2 UserDetails

`UserDetails` က Spring Security သိတဲ့ user information format ပါ။

သူ့ထဲမှာ usually ဒီအချက်တွေပါနိုင်တယ်။

- username/email
- password
- roles/authorities
- account expired ဖြစ်လား
- account locked ဖြစ်လား

---

### 6.3 UserDetailsService

`UserDetailsService` က database ထဲက user ကို load လုပ်ဖို့သုံးတယ်။

```java
@Service
public class CustomUserDetailsService implements UserDetailsService {

    @Override
    public UserDetails loadUserByUsername(String username) {
        // database ထဲမှာ user ရှာ
        return userRepository.findByEmail(username)
            .orElseThrow(() -> new UsernameNotFoundException("User not found"));
    }
}
```

---

### 6.4 PasswordEncoder

Password ကို plain text မသိမ်းသင့်ပါ။

ဒီလိုမသိမ်းသင့်:

```text
password = 123456
```

ဒီလို hash လုပ်ပြီး သိမ်းသင့်:

```text
$2a$10$asdjklasdjklasdjkl...
```

Spring Security မှာ common သုံးတာက `BCryptPasswordEncoder` ပါ။

```java
@Bean
public PasswordEncoder passwordEncoder() {
    return new BCryptPasswordEncoder();
}
```

## 7. JWT ဆိုတာဘာလဲ?

JWT = **JSON Web Token**

Login ဝင်ပြီးရင် server က user ကို token ပေးတယ်။

အလွယ်ဆုံးပြောရရင်:

```text
JWT = Login ဝင်ပြီးရင်ရတဲ့ ID card
```

---

## 8. JWT Authentication Flow

```text
1. User login လုပ်တယ်
   email/password ပို့တယ်

2. Server က user ကိုစစ်တယ်
   password မှန်လားစစ်တယ်

3. မှန်ရင် JWT token ထုတ်ပေးတယ်

4. Client က token သိမ်းထားတယ်

5. Next API request တိုင်းမှာ token ထည့်ပို့တယ်

6. Server က token စစ်တယ်

7. Token မှန်ရင် API access ပေးတယ်
```

---

## 9. JWT Example

Login response:

```json
{
  "token": "eyJhbGciOiJIUzI1NiJ9..."
}
```

Next API request မှာ:

```http
Authorization: Bearer eyJhbGciOiJIUzI1NiJ9...
```

Meaning:

```text
ငါ login ဝင်ပြီးသားပါ။ ဒီ token နဲ့စစ်ပေးပါ။
```

---

## 10. JWT Filter ဆိုတာဘာလဲ?

JWT Filter က request တိုင်းမှာ token ကိုစစ်တဲ့ class ပါ။

Flow:

```text
Request ရောက်လာတယ်
      ↓
Authorization header ရှိလားစစ်
      ↓
Bearer token ပါလားစစ်
      ↓
Token မှန်လားစစ်
      ↓
မှန်ရင် SecurityContext ထဲ user ထည့်
      ↓
Controller ဆီသွား
```

Example logic:

```java
String authHeader = request.getHeader("Authorization");

if (authHeader != null && authHeader.startsWith("Bearer ")) {
    String token = authHeader.substring(7);

    String username = jwtService.extractUsername(token);

    if (username != null && SecurityContextHolder.getContext().getAuthentication() == null) {
        UserDetails userDetails = userDetailsService.loadUserByUsername(username);

        if (jwtService.isTokenValid(token, userDetails)) {
            UsernamePasswordAuthenticationToken authToken =
                new UsernamePasswordAuthenticationToken(
                    userDetails,
                    null,
                    userDetails.getAuthorities()
                );

            SecurityContextHolder.getContext().setAuthentication(authToken);
        }
    }
}
```

## 11. Stateless Authentication ဆိုတာဘာလဲ?

JWT သုံးတဲ့ API တွေမှာ server က session မသိမ်းဘူး။

```text
Server က user login state ကို memory ထဲမသိမ်းဘူး
Client က token ပို့လာမှ token ကိုစစ်တယ်
```

ဒါကို **stateless authentication** လို့ခေါ်တယ်။

Security config မှာ:

```java
.sessionManagement(session -> session
    .sessionCreationPolicy(SessionCreationPolicy.STATELESS)
)
```

## 12. SecurityContext ဆိုတာဘာလဲ?

`SecurityContext` က current request ထဲမှာ authenticated user information သိမ်းတဲ့နေရာပါ။

ဥပမာ:

```java
Authentication authentication = SecurityContextHolder.getContext().getAuthentication();
```

JWT token မှန်ရင် Spring Security က authenticated user ကို ဒီနေရာထဲထည့်ထားတယ်။

## 13. RBAC ဆိုတာဘာလဲ?

RBAC = **Role-Based Access Control**

Role အလိုက် API access ခွင့်ပြုတာပါ။

ဥပမာ:

```text
USER  → profile ကြည့်နိုင်
ADMIN → user manage လုပ်နိုင်
```

အလွယ်မှတ်ရန်:

```text
RBAC = Role အလိုက် permission စစ်တာ
```

---

## 14. Role Example

Common roles:

```text
ROLE_USER
ROLE_ADMIN
ROLE_MANAGER
```

Spring Security မှာ role တွေကို usually `ROLE_` prefix နဲ့သုံးတယ်။

## 15. RBAC Annotation များ

Spring Security မှာ method level security အတွက် commonly သုံးတဲ့ annotation တွေရှိတယ်။

အရေးကြီးဆုံးတွေက:

1. `@PreAuthorize`
2. `@Secured`
3. `@RolesAllowed`

---

## 16. @PreAuthorize

`@PreAuthorize` က method မဝင်ခင် permission/role စစ်တယ်။

ဒါက recommended အများဆုံး annotation ပါ။

### ADMIN only

```java
@PreAuthorize("hasRole('ADMIN')")
@GetMapping("/admin/users")
public List<User> getAllUsers() {
    return userService.getAllUsers();
}
```

Meaning:

```text
ADMIN role ရှိတဲ့ user ပဲ ဒီ API ကိုခေါ်လို့ရမယ်။
```

---

### USER or ADMIN

```java
@PreAuthorize("hasAnyRole('USER', 'ADMIN')")
@GetMapping("/profile")
public UserProfile getProfile() {
    return userService.getProfile();
}
```

Meaning:

```text
USER ဖြစ်ဖြစ် ADMIN ဖြစ်ဖြစ် access ရမယ်။
```

---

### Permission condition

```java
@PreAuthorize("hasAuthority('USER_DELETE')")
public void deleteUser(Long id) {
    // delete user
}
```

Meaning:

```text
USER_DELETE permission ရှိမှ delete လုပ်လို့ရမယ်။
```

---

### Own data only

```java
@PreAuthorize("#userId == authentication.principal.id")
@GetMapping("/users/{userId}")
public User getUser(@PathVariable Long userId) {
    return userService.findById(userId);
}
```

Meaning:

```text
User က သူ့ data ပဲကြည့်ခွင့်ရှိမယ်။
```

---

## 17. @Secured

`@Secured` က simple role check အတွက်သုံးတယ်။

```java
@Secured("ROLE_ADMIN")
@GetMapping("/admin/dashboard")
public String dashboard() {
    return "Admin dashboard";
}
```

Important:

```text
@Secured မှာ ROLE_ prefix ထည့်ရေးရတတ်တယ်။
```

---

## 18. @RolesAllowed

`@RolesAllowed` က Java standard annotation style ပါ။

```java
@RolesAllowed("ADMIN")
@GetMapping("/admin/reports")
public String reports() {
    return "Reports";
}
```

---

## 19. @PreAuthorize vs @Secured vs @RolesAllowed

| Annotation | Usage | Flexibility |
|---|---|---|
| `@PreAuthorize` | Modern Spring Security | အမြင့်ဆုံး |
| `@Secured` | Simple role check | Medium |
| `@RolesAllowed` | Java standard style | Medium |

Interview မှာပြောလို့ကောင်းတာ:

```text
I prefer @PreAuthorize because it is more flexible and supports expressions.
```

## 20. Method Security Enable လုပ်ဖို့လိုတယ်

RBAC annotation တွေအလုပ်လုပ်ဖို့ config class မှာ `@EnableMethodSecurity` ထည့်ဖို့လိုတယ်။

```java
@Configuration
@EnableMethodSecurity
public class SecurityConfig {
    // security config
}
```

`@EnableMethodSecurity` မထည့်ရင် `@PreAuthorize`, `@Secured` တွေ မအလုပ်လုပ်နိုင်ပါ။

---

## 21. URL Level Security vs Method Level Security

### URL Level Security

Security config ထဲမှာ URL path အလိုက်စစ်တာ။

```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/auth/**").permitAll()
    .requestMatchers("/api/admin/**").hasRole("ADMIN")
    .anyRequest().authenticated()
)
```

---

### Method Level Security

Controller/Service method ပေါ်မှာ annotation နဲ့စစ်တာ။

```java
@PreAuthorize("hasRole('ADMIN')")
public void deleteUser() {
    // admin only
}
```

## 22. Real Project Example

API structure:

```text
POST /api/auth/register      → public
POST /api/auth/login         → public
GET  /api/me                 → USER or ADMIN
GET  /api/friends            → USER or ADMIN
DELETE /api/admin/users/{id} → ADMIN only
```

Security rule:

```java
.authorizeHttpRequests(auth -> auth
    .requestMatchers("/api/auth/**").permitAll()
    .requestMatchers("/api/admin/**").hasRole("ADMIN")
    .anyRequest().authenticated()
)
```

Method security:

```java
@PreAuthorize("hasAnyRole('USER', 'ADMIN')")
@GetMapping("/api/me")
public UserProfile me() {
    return userService.getCurrentUser();
}

@PreAuthorize("hasRole('ADMIN')")
@DeleteMapping("/api/admin/users/{id}")
public void deleteUser(@PathVariable Long id) {
    userService.deleteUser(id);
}
```

## 23. Full JWT + RBAC Flow

```text
1. User login
      ↓
2. Server checks email/password
      ↓
3. Password correct
      ↓
4. Server generates JWT token with user role
      ↓
5. Client stores token
      ↓
6. Client calls protected API with token
      ↓
7. JWT filter validates token
      ↓
8. Spring Security sets user in SecurityContext
      ↓
9. @PreAuthorize checks role
      ↓
10. Role correct → API access allowed
      ↓
11. Role wrong → 403 Forbidden
```

| 401 Unauthorized | Login မဝင်ထား / token မမှန် |
| 403 Forbidden | Login ဝင်ထားပေမယ့် permission မရှိ |

ဥပမာ:

```text

USER role နဲ့ ADMIN API ခေါ်တယ် → 403
```

## 25. Common Mistakes

### Mistake 1: Password plain text သိမ်းတာ

မလုပ်သင့်:

```text
password = 123456
```

လုပ်သင့်:

```text
password = BCrypt hash
```

---

### Mistake 2: @EnableMethodSecurity မထည့်တာ

Annotation တွေ မအလုပ်လုပ်နိုင်ပါ။

---

### Mistake 3: ROLE_ prefix ရောထွေးတာ

`hasRole('ADMIN')` ဆို Spring က internally `ROLE_ADMIN` လို့ယူတယ်။

```java
@PreAuthorize("hasRole('ADMIN')")
```

ဒါက usually correct ဖြစ်တယ်။

`@Secured` မှာတော့:

```java
@Secured("ROLE_ADMIN")
```

---

### Mistake 4: JWT expiration မထည့်တာ

JWT token ကို expire time ထည့်သင့်တယ်။

ဥပမာ:

```text
Access token expire = 15 minutes / 1 hour
Refresh token expire = 7 days / 30 days
```

## 27. Interview Short Answer

### What is Spring Security?

Spring Security is a framework that provides authentication, authorization, and protection for Spring applications.

### What is Authentication?

Authentication verifies who the user is, usually by checking username and password.

### What is Authorization?

Authorization checks what the authenticated user is allowed to do.

### What is JWT?

JWT is a token used for stateless authentication. After login, the server returns a token, and the client sends it with every request.

### What is RBAC?

RBAC means Role-Based Access Control. It controls API access based on user roles such as USER and ADMIN.

### What is @PreAuthorize?

`@PreAuthorize` checks role or permission before executing a method.
